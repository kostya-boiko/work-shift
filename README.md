## Commit Message Format

\<type>: short description in present tense, lowercase>

Types:


- feat — new feature

- fix — bug fix

- refactor — code change without functionality change

- style — code style / formatting / CSS/SCSS

- docs — documentation changes

- test — adding or modifying tests

- chore — minor tasks / maintenance / dependency updates

- perf — performance improvements

- ci — changes to CI/CD configuration

- build — build system / dependencies affecting build

- revert — revert previous commit


Examples:

- feat: add timer for drawing task

- fix: prevent empty category submission

- refactor: simplify image fetch logic

- style: adjust button padding

- docs: update readme with setup instructions

- test: add unit tests for image component

- chore: update npm packages

- perf: optimize image loading

- ci: update workflow for tests

- build: update webpack config

- revert: feat: add timer for drawing task
